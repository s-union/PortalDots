# Contributing

Issues and pull requests are welcome!

## Feature requests (Discussions)

Open a new discussion in the [Ideas category](https://github.com/s-union/PortalDots/discussions/new?category=Ideas). A [GitHub account](https://github.com/signup) is required.

## Bug reports (Issues)

Please [open an issue](https://github.com/s-union/PortalDots/issues/new). Any format is fine — the more context you provide, the faster we can respond. A [GitHub account](https://github.com/signup) is required.

You are also welcome to submit a pull request directly without opening an issue first. Note that PRs may be closed without merging depending on the scope of the change. Opening an issue first is recommended for anything beyond typo fixes or obvious bug fixes.

## Submitting a pull request

- Target the `main` branch.
- Please [discuss with the maintainers](https://github.com/s-union/PortalDots/discussions/new?category=q-a) before submitting a PR that:
    - Adds a new migration file
    - Makes significant UI or copy changes (beyond typos and minor fixes)

### Development workflow

1. Fork and clone the repository.
2. Create a branch from `main` (naming convention: `feature/{issue-number}_{snake_case_description}`).
    - e.g. `feature/13_update_readme`
3. Make your changes.
4. Submit a pull request.

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.
